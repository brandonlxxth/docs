This guide covers the basic workflow for creating, playing, and reviewing games in Break & Run.

## Creating a new game

### Standard modes (Race to, Sets of, Best of, Free play)

1. Tap **New Game** from the home screen
2. Enter player names
3. Select a game mode
4. Configure game settings:
   - **Race to**: Set target score
   - **Sets of**: Set target score and number of sets
   - **Best of**: Set target score and number of sets (must be odd)
   - **Free play**: No configuration needed
5. Choose who breaks first
6. Tap **Start Game**

### Killer mode

1. Tap **New Game** from the home screen
2. Select **Killer** mode
3. Add players (minimum 3):
   - Enter player name
   - Tap **Add Player**
   - Repeat for all players
4. Set starting lives for each player
5. Optionally enable **Trickshot Black Ball** rule
6. Choose who breaks first
7. Tap **Start Game**

## During gameplay

### Tracking scores

Use the scoreboard buttons to track game progress:

- **+1**: Award a frame to the player
- **Dish**: Record a dish (break dish or reverse dish)
- **-1**: Deduct a frame (for corrections)
- **Miss**: Track a missed shot (Killer mode)

### Dish types

When recording a dish, select the type:
- **Break dish**: Player dishes on the break
- **Reverse dish**: Player dishes after opponent's dish
- **Miss**: Player misses (standard tracking)
- **Trickshot black**: Attempt to pot black ball (Killer mode only)

### Ball color selection

For standard modes, assign ball colors:
- Tap **Red** or **Yellow** to assign colors
- Select **Foul Break** if applicable
- Colors persist throughout the game

### Killer mode specifics

- **Player selection**: Animated indicator shows current player
- **Lives display**: Each player's remaining lives shown in real-time
- **Elimination**: Players with 0 lives are marked as eliminated
- **Trickshot black**: 
  - Success: Player gains a life
  - Failure: Player loses a life

## Game controls

### Pause or exit
- Tap the back button to pause
- Game auto-saves on each frame
- Resume from home screen if interrupted

### Undo last action
- Use the undo button to reverse the last score change
- Frame history is updated accordingly

### End game early
- Tap **End Game** to finish before reaching target
- Winner is determined by current scores
- Game is saved to history

## Viewing game history

### Past games screen

1. Tap **Past Games** from home screen
2. Browse completed games
3. Tap any game to view details

### Game details

View comprehensive game information:
- **Summary**: Players, scores, winner, date/time
- **Frame history**: Frame-by-frame breakdown with timestamps
- **Set history**: Set-by-set results (Sets of/Best of modes)
- **Statistics**: Dishes, score progression
- **Killer standings**: Final player rankings (Killer mode)

### Filtering and search

- Search by player name
- Filter by game mode
- Sort by date
- View head-to-head statistics

## Resuming games

If you exit during an active game:

1. Return to home screen
2. Look for **Resume Game** section
3. Tap **Resume** to continue
4. All game state is preserved:
   - Current scores
   - Frame history
   - Player colors
   - Set progress
   - Killer player lives

Track performance over time:

## Tips and best practices

### Accurate tracking
- Record frames immediately after they occur
- Use dish types consistently
- Verify scores before ending game
- Utilize undo for mistakes

### Killer mode
- Clearly communicate whose turn it is
- Track lives carefully
- Decide trickshot attempts before breaking
- Confirm eliminations with all players

### Game management
- Name players consistently for better statistics
- End games properly (don't force quit)
- Review history to settle disputes
- Use Free Play mode for practice

### Cloud sync (web version)
- Create an account to sync across devices
- Games auto-save to cloud
- Access history from any device
- Backup is automatic

## Troubleshooting

### Game not saving
- Ensure you're recording at least one frame
- Check storage permissions (Android)
- Verify internet connection (web version with cloud sync)

### Resume not working
- Game must have at least one frame recorded
- Check if game was properly started
- Verify app wasn't force-closed during critical save

### Statistics incorrect
- Ensure player names are spelled consistently
- Check for duplicate player entries
- Review game history for anomalies

### Killer mode issues
- Verify all players are added before starting
- Ensure lives are set correctly
- Check trickshot rule is enabled if using

## Keyboard shortcuts (web version)

- **Space**: Award frame to current player
- **D**: Record dish
- **U**: Undo last action
- **Esc**: Pause game
- **Enter**: Confirm dialog actions
