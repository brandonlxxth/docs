## Overview

Pay at Table lets you **generate QR codes** that customers scan at the table to **view their bill and pay by credit card** without going to the till.

## Required permissions

| Feature | User Role Permission | Tab in User Role Config |
|---------|---------------------|------------------------|
| Pay at Table | QR Code Generator | Configuration (Pay At Table) |

**What you do:** Create **QR code configurations** in the Management Portal (name, outlet, payment account, optional logo, colour, gratuity). Each config has a base URL. When you **view** a saved config, you enter a **table number** (1–9999) to generate a QR that deep-links to that table’s order, or use **bulk download** (start/end table, up to 100) to get a ZIP of QR images.

**What the customer does:** Scans the QR. If the URL includes a table number they go straight to that table’s bill; otherwise they see a **table selection** screen, then the order page. They check the bill and pay by card (**Stripe Checkout** or **JudoPay**), with optional **gratuity** if you configured it. Payment applies to that table’s order; the till can show alerts and **print a receipt** if configured.

**Context:** Works with **network selling master terminals** and **table service** where operators manage table tabs via **Table Layout** on the till.

<Warning>
**Requires Pay at Table module**: This feature requires the Pay at Table module, which is charged separately. If you don't see this section in your navigation, contact your administrator or [Kappture Sales](/contact) to enable the Pay at Table module.
</Warning>

**Permissions:** You need the **Pay at Table QR Code Generator** permission to access this page.

## How users use Pay at Table

### Staff and venue setup

1. **Enable the module** — Ensure the Pay at Table module is enabled (see warning above).

2. **Create a QR code configuration** — **Configuration > Management > Pay at Table > QR Code Generator** → click **+**.
   - **Name** (e.g. "Restaurant", "Zone A").
   - **Outlet** (only outlets with at least one order terminal are listed).
   - **Account** — **Stripe Checkout** or **JudoPay**; required for card payment.
   - **Logo** (optional) — Resource type Logo; **Color** (optional).
   - **Gratuity Option 1, 2, 3** (optional) — 0–100% tip presets.
   - Save.

3. **Generate QR codes per table** — After saving, click **View** on the config.
   - **Single** tab: Enter **Table number** (1–9999). URL updates; QR image updates. Use **Copy** or **Download** (PNG).
   - **Bulk** tab: **Start** and **End** table number (max 100). Click **Generate & Download ZIP** → ZIP of table-1.png, table-2.png, …

4. **Print or display** — Print QR codes (e.g. table tents, stickers) or display on screens at each table.

5. **Terminal configuration** — On terminals used for table service: enable **Pay at Table**, set **Tender** (e.g. Card/EFT), optionally **Print receipt**. Compatible with **network selling master terminals**; operators use **Table Layout** on the till.

### Customer flow

1. **Scan** — Customer scans the QR on the table with their smartphone (camera or QR app).

2. **Next step** — URL has table number → go straight to that table’s order page. No table number → **table selection** screen; customer enters or selects table.

3. **View bill** — Payment flow loads the order for that outlet and table. Customer sees items and total.

4. **Pay** — Pay by **credit card** (Stripe or JudoPay). Optional tip if gratuity options are set. Payment applies to that table’s order; till shows alerts and can print receipt if enabled.

## Using Pay at Table (Management Portal)

### Accessing Pay at Table

Navigate to **Configuration > Management > Pay at Table > QR Code Generator**.

### Creating a QR code configuration

List of configs → click **+**. In the form:

- **Name** — Display name (e.g. "Restaurant", "Zone A").
- **Outlet** — Config applies to this outlet; only outlets with at least one order terminal appear.
- **Account** — **Stripe Checkout** or **JudoPay**; required for card payment.
- **Logo** — Optional; resource type **Logo** from **Configuration > Management > Resources**.
- **Color** — Optional; payment screen colour.
- **Gratuity Option 1, 2, 3** — Optional; 0–100% (e.g. 10, 15, 20).

Save. Base URL is generated; table number is added when you generate individual QR codes.

### Viewing, generating QR codes, and bulk download

Select a config → **View**. Modal with two tabs:

- **Single** — Enter **Table number** (1–9999). URL and QR image update. **Copy** URL or **Download** PNG.
- **Bulk** — **Start** and **End** table number (1–9999, max 100). **Generate & Download ZIP** → one PNG per table.

### Editing and deleting

**Edit** → change Name, Outlet, Account, Logo, Color, gratuity → Save. **Delete** when no longer in use (existing printed QR links will stop working).

## Pay at Table fields reference

| Field | What it does |
|-------|----------------|
| **Name** | Display name for the config (e.g. Restaurant, Zone A). |
| **Outlet** | Config applies to this outlet; only outlets with an order terminal listed. |
| **Account** | **Stripe Checkout** or **JudoPay**; required for card payment. |
| **Logo** | Optional; resource type **Logo** (Resources); payment screen. |
| **Color** | Optional; payment screen colour. |
| **Gratuity Option 1, 2, 3** | Optional; tip % (0–100%) when paying. |
| **Table number** (in View) | When viewing: enter 1–9999 to generate QR for that table. URL: `…/order/{orgId}/{codeId}/{tableNumber}`. |
| **Bulk: Start / End** | View > Bulk: range (max 100); downloads as ZIP. |

### Terminal configuration (Pay at Table)

| Setting | What it does |
|--------|----------------|
| **Pay at Table** | Enable on terminal; participates in Pay at Table orders and shows alerts. |
| **Tender** | Tender used for Pay at Table payment (e.g. Card/EFT). |
| **Print receipt** | Till prints receipt when Pay at Table payment completes. |

## How Pay at Table links

- **Outlets and terminals** — QR config tied to **Outlet** with an order terminal. Table numbers (1–9999) align with **Table Layout** and till.
- **Account** — Card payment via **Stripe Checkout** or **JudoPay** on the QR config. Configure accounts under **Configuration**.
- **Table Layout** — Operators use it on the till for table tabs and orders. Pay at Table QR links customers to that table’s bill; table number must match till.
- **Terminals** — Enable **Pay at Table**, set **Tender**, **Print receipt**. Works with **network selling master terminals**.
- **Resources** — **Logo** on QR config is resource type **Logo** from **Configuration > Management > Resources**.

## Best practices

- Clear **Name** per config so staff know which to use.
- At least one **order terminal** on the outlet so it appears in the list.
- **Single** tab for one QR per table; **Bulk** for up to 100 in one ZIP.
- Table numbers (1–9999) consistent with **Table Layout** and till.
- On table-service terminals: **Pay at Table** on, **Tender** set, **Print receipt** if desired.
- Place QR codes where customers can scan easily (table tents, stickers, screens).
