## Overview

Under **Configuration > Management > Messaging** you can:

- **Send Message Templates** — Create **server message templates** used for system notifications (e.g. server push messages to terminals or other systems).
- **Terminal Message Templates** — Create **terminal message templates** used when sending messages to terminals (e.g. "Please see manager", "Break in 10 minutes"). You choose a template when sending a message to POS.
- **Send Message to POS** — **Send messages directly to terminals** in real time. You select one or more terminals (or outlets), choose a **Terminal Message Template** or enter custom text, and send. The message appears on the selected terminals.
- **Message History** — View **message history** and delivery status for messages sent to terminals (who sent what, when, to which terminals).

This is separate from **Configuration > Campaign** (Messaging and Notification templates), which is for **member** messaging (push and email to members). Management Messaging is for **operator and terminal** communication.

You need the **Server Message Template**, **Terminal Message Template**, or **Terminal Send Message** permission as appropriate for each page.

## Using Management Messaging

### Accessing Messaging (Management)

Navigate to **Configuration > Management > Messaging** in the navigation menu. The submenu has:

- **Send Message Templates** — Create and edit server message templates.
- **Terminal Message Templates** — Create and edit terminal message templates.
- **Send Message to POS** — Send a message to terminals now.
- **Message History** — View history of messages sent to terminals.

### Send Message Templates

When you open **Configuration > Management > Messaging > Send Message Templates**, you see a list of server message templates. Click the **+** icon to create a new template. Enter a **Name** and the **message content** (and any variables or options as shown). Save. Server message templates are used by the system for server-initiated notifications to terminals or other systems.

### Terminal Message Templates

When you open **Configuration > Management > Messaging > Terminal Message Templates**, you see a **selector at the top** and a **+** icon. Choose a template from the list; its details load below. Create or edit templates with a **Name** and **message content**. When you send a message to POS (Send Message to POS), you can choose one of these templates so the same message can be reused (e.g. "Please see manager", "Break in 10 minutes").

### Send Message to POS

When you open **Configuration > Management > Messaging > Send Message to POS**, you see a form to send a message to terminals. Select the **terminals** (or outlets) you want to send to, choose a **Terminal Message Template** or enter **custom text**, and click Send. The message is delivered to the selected terminals in real time and appears on the till screen (or in the terminal inbox, depending on configuration).

### Message History

When you open **Configuration > Management > Messaging > Message History**, you see a list or table of messages sent to terminals: who sent the message, when, which terminals received it, and delivery status. Use it to confirm delivery and to review past communications.

## How Management Messaging links

- **Terminals**: Messages are sent to **Terminals** (or to all terminals at selected outlets). Operators see the message on the till.
- **Terminal Message Templates**: Templates are used when sending from **Send Message to POS** so you don’t have to retype common messages.
- **Server Message Templates**: Used by the system for server-initiated notifications (e.g. system alerts, sync messages).

## Best practices

- Create **Terminal Message Templates** for common messages (e.g. "Please see manager", "Break in 10 minutes") so staff can send them quickly from Send Message to POS.
- Use **Message History** to verify that important messages were delivered.
- Do not confuse this with **Configuration > Campaign > Messaging** (member messaging); Management Messaging is for operator/terminal communication only.
